# Brainblast v0.8.1 — Signguard

**Keyguard protects the keypair from deletion. Signguard protects it from being *used against you*.**

The single most common way SOL actually leaves a wallet is signing one transaction you didn't
understand — a drainer, a `SetAuthority`, a delegate `Approve`, a runaway transfer. And agents now
sign transactions autonomously. Signguard is the transaction-signing sibling of the file Guard: a
standing **local signing policy** that decodes a transaction *before it's signed* and returns
`allow / warn / block`.

It builds directly on the shipped `firewall` (decode + pattern findings + simulation) and adds the
thing the firewall doesn't: **a policy you set once and enforce everywhere.**

## What Signguard adds on top of the firewall

1. **Spend quantification + caps** — the firewall flags patterns but never says *how much leaves*.
   Signguard decodes SystemProgram `Transfer`/`CreateAccount` lamports out of the fee payer, sums the
   **SOL leaving**, and enforces a **per-transaction cap** and a **cumulative per-session cap**.
   "This moves 41.2 SOL — over your 1 SOL limit. BLOCKED."
2. **Program allowlist** — unknown programs become a hard **block** (not the firewall's soft warn),
   scoped to `KNOWN_PROGRAMS` ∪ your policy's `allowedPrograms`.
3. **Action policy** — `setAuthority` / `delegateApproval` / `programUpgrade` / `closeAccount` each map
   to `allow|warn|block` (secure defaults: the first three block, close warns).
4. **Recipient allowlist** (optional) — SOL/token transfers must go to addresses you've approved.
5. **Session ledger** — cumulative spend tracked at `~/.brainblast/signguard/session.json`, with
   `signguard reset`.

## Surfaces

- **`brainblast signguard <base64-tx>`** — decode + simulate (reuse firewall) + apply policy; exit 1 on
  block. `--policy`, `--max-sol`, `--record`, `--no-sim`, `--json`.
- **`brainblast signguard init`** — scaffold a secure-default policy at `.brainblast/signguard.json`.
- **`brainblast signguard hook`** — PreToolUse entrypoint: parses recognized money/authority Bash
  commands (`solana transfer`, `spl-token transfer`, `solana program deploy`/`set-upgrade-authority`)
  and applies the policy; deny/ask JSON.
- **`inspectSigning(base64, { policy })`** — inline export for agent frameworks to call before signing
  (the firewall's proven inline pattern, now policy-aware).
- **`signguard session` / `signguard reset`** — view / clear the cumulative ledger.

## Build order

1. `policy.ts` — types, secure defaults, load/scaffold.
2. `transfers.ts` — quantify SOL out (and list token transfers + recipients) from a `DecodedTx`.
3. `evaluate.ts` — the policy engine over `DecodedTx` + firewall findings + session.
4. `session.ts` — the cumulative ledger.
5. `index.ts` — `inspectSigning` orchestration + render.
6. CLI + hook + tests.

Each piece is pure/injectable for offline tests (the firewall decoder + RPC sim are already
fetch-injectable). Ships as a point release: `0.8.0 → 0.8.1`.
