# Brainblast v0.9.1 — Tier 2: the context-scaled sandbox

v0.9.0 shipped the pluggable oracle and the offline tiers (static + compiler).
`executed-test` and `differential` were wired but **abstained** — they run candidate
code, which is a security question, so they waited for the sandbox. v0.9.1 ships it.

## What shipped

- **`src/oracle/sandbox.ts`** — `runInSandbox(spec)`, isolation scaled to
  `target.context`:
  - **`local` → light isolate**: child process (`spawnSync`) in an ephemeral dir,
    hard wall-clock timeout (kills the child), output-size cap, trimmed env. Stops
    a runaway test on the user's OWN code; not a malice defense, no network-
    isolation claim.
  - **`ingest` → hardened container**: `docker/podman run --rm --network=none
    --read-only --tmpfs /tmp --memory --cpus --pids-limit --cap-drop=ALL
    --security-opt=no-new-privileges --user nobody`. **Refuses (`status:"refused"`
    → UNKNOWN) when no runtime/daemon is available — never falls back to the light
    isolate.** `containerRuntime()` is the cheap, no-exec probe.
  - Crash / timeout / refusal → **UNKNOWN, never RED**.
- **`executedTestBackend`** (Tier 2, default OFF) — `renderTest(rule.test.kind, …)`
  (a vetted template) → run in the sandbox vs the candidate → RED iff the contract
  fails. Proven RED→GREEN on `stripe-webhook-signature`.
- **`differentialBackend`** (Tier 2, default OFF) — new `check.kind:
  "differential-io"`: run ONLY the candidate against a vetted golden I/O table in
  the sandbox → RED iff any output diverges. Golden-I/O flavor first (candidate-
  only execution); live-reference is a follow-up.
- **`solana-lamports-scaling-wrong-constant`** pack — SOL→lamports with `1e6`
  instead of `1e9`. A `wrong-constant` logic bug with no static signature, proven
  RED→GREEN by the differential oracle — closing the uncovered class.
- **Wiring** — `pack validate` reports `differential-io` rules as non-fatal
  `unverifiable` on the offline path; `brainblast verify <pack> --oracle=executed|
  differential` proves them (opt-in). `runInSandbox` / `containerRuntime` exported.

## Tests (553 pass, 1 Docker-gated skip)

- Light isolate kills an infinite loop → `timeout`; output cap contains a print
  bomb (`ENOBUFS` → `error`, bounded output).
- Ingest **refuses** (never light-isolates) when no container runtime exists — the
  property that actually matters under adversarial input — and is always tagged
  `hardened`.
- `executed-test` proves RED→GREEN on the Stripe webhook contract through the
  sandbox; `differential` proves RED→GREEN on the wrong-constant pack; a hanging
  candidate is UNKNOWN, never RED.
- Hostile-fixture containment (network / out-of-dir) runs where Docker exists,
  skips cleanly otherwise — honest, not faked.

## Discipline kept

The default `npx brainblast` executes no candidate code (byte-for-byte offline);
an executed/differential RED only counts when the test/reference is a vetted, owned
template; UNKNOWN counts as GREEN for gating but is never a proof; and the local
isolate makes no network-isolation claim — only the hardened ingest container gets
`--network=none`.
