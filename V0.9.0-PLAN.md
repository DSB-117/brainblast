# Brainblast v0.9.0 — The Generalized Oracle

Turn `runChecker` from one static pattern-matcher into a **pluggable verification
interface**, so the corpus can grow past hand-written Solana checkers into the long
tail of agent errors — without changing the RED→GREEN contract, and while keeping
the cheap offline path the default.

## Thesis: one verdict, many oracles

A trap is reward-gradable because a **deterministic procedure** returns RED on the
vulnerable code and GREEN on the fixed code, runnable by anyone, with no secret
answer key. v0.9.0 makes the *procedure* pluggable while keeping the *verdict*
identical (`RED | GREEN | UNKNOWN`). Every backend answers the same question —
"does this code ship the trap?" — and is tried in trust order.

| What makes the code wrong | Backend | Tier |
|---|---|---|
| A forbidden code shape is present | `static-checker` (default) | 0 |
| Doesn't type-check against the pinned SDK | `compiler` | 1 |
| A behavioral contract is violated at runtime | `executed-test` | 2 |
| Output diverges from a known-correct reference | `differential` | 2 |

## What shipped in v0.9.0 (Tiers 0–1, the whole thesis)

- **`src/oracle/types.ts`** — the `OracleBackend` interface, `OracleVerdict`,
  `OracleTarget` (with `context: "local" | "ingest"`), method/tier enums.
- **`src/oracle/backends/staticChecker.ts`** — wraps `auditWithRule` (Tier 0). The
  seam is a no-op, proved by a parametrized test over every bundled pack.
- **`src/oracle/backends/compiler.ts`** — Tier 1. Type-checks the candidate against
  the pinned SDK with `ts-morph` (compiler only, never the program). New checker
  kind `compiles-against-sdk`. Offline, deterministic, no execution.
- **`src/oracle/prove.ts`** — the one true gate: `proveRedGreen`, `proveWithBest`,
  `proofMethod`. Records whatever actually proved RED→GREEN (incl. compound,
  e.g. `static-checker+compiler`).
- **`src/oracle/index.ts`** — backend registry, tier-gated `selectBackends`,
  `parseOracleSelector`, inline `auditWithOracle`.
- **Surfaces** — `brainblast verify <pack-dir> [--oracle=…]` (reproduction
  scorecard); `--oracle` on the main audit (additive, advisory); oracle-aware
  `pack validate`; MCP `brainblast_verify`.
- **Off-Solana demo** — bundled pack `stripe-paymentintents-moved`, compiler-proven
  RED→GREEN on `stripe@17` with zero code execution.

## What is deferred to v0.9.1 (Tier 2)

`executed-test` and `differential` ship the **interface, tier, `supports()`, and an
honest refusal** — they run candidate code, so they are opt-in only and abstain
(UNKNOWN, never RED) until the **context-scaled sandbox** lands: a light
worker/`vm` isolate for the user's own code locally, and a hardened container
(`--network=none`, read-only, dropped caps, resource limits) that **refuses rather
than falls back** for contributor code on the ingest path. The interface is
identical either way, so nothing downstream changes when it lands.

## Discipline kept

- The RED→GREEN contract is unchanged; only the number of ways to establish it grew.
- Rules stay pure data binding to **vetted templates**; no backend executes
  contributor-authored logic as an oracle.
- The default `npx brainblast` stays offline, deterministic, LLM-free, and executes
  **no candidate code** — byte-for-byte the 0.8.3 cost path.
- UNKNOWN counts as GREEN for *gating* but is **never** a valid proof on a record.
