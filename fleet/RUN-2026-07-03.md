## Honesty gate — what did NOT land

- **3 fabricated candidates rejected.** The coinbase/agentkit scout admitted the
  forbidden literals were **not present in the repo** — it "synthesized" plausible
  regressions. Deleted: their provenance would have falsely claimed agentkit ships
  them. The proof gate would have passed them (the checker can't tell invented code
  from real), so this rejection was a manual honesty call.
- **3 redundant skipPreflight duplicates pruned** (Raydium volume bot, Shyft,
  quiknode `sendRawTransaction`) — kept only the two distinct call surfaces so the
  corpus doesn't become a skipPreflight monoculture.

## Why the clean repos were clean

Polished SaaS templates (saas-starter, open-saas, next-forge, skateshop,
yournextstore, react-starter-kit) verify webhook signatures, pin `apiVersion`, and
set `httpOnly/secure/sameSite` safely. Well-maintained Solana apps (eliza, neur-app,
solana-tools, nft-armory) use `skipPreflight: false`, `commitment: 'confirmed'`, and
non-zero slippage defaults. Real footguns cluster in older / lower-polish code and
tutorial/bot repos.

## Recurring footgun shapes the current checker can't express

Candidate checker proposals for a future run (Move 2):

- **Absent-property** algorithm confusion — `expressjwt({ secret })` with no
  `algorithms` (express-mongoose-es6-rest-api). Omission, not a forbidden literal.
- **Absent-argument** no-expiry tokens — `jwt.sign(payload, secret)` with no `expiresIn`.
- **Missing-call** unconfirmed state — `sendRawTransaction(...)` returned with no
  `confirmTransaction` (neur-app).
- **Nested-property** insecure cookies — `session({ cookie: { secure: false } })`;
  the flag is nested under `cookie`, unreachable by the flat checker. (This is why
  express-session was discovered but not scouted this run.)
