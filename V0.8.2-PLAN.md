# Brainblast v0.8.2 — Wallet Guard

**Catch the silent wallet-adapter footguns that break demos, ship the wrong cluster, and leak paid RPC keys.**

The trigger: a devnet demo whose `.env` said `NEXT_PUBLIC_SOLANA_NETWORK=devnet`, but neither agent
wired it into the wallet adapter — so the `ConnectionProvider` ran mainnet, the wallet referenced real
SOL, and the demo silently behaved on the wrong network. Classic, common, and exactly the silent
config mismatch Brainblast exists to surface.

Wallet Guard is one analysis pass over the wallet-adapter setup (`@solana/wallet-adapter-react`) + the
project's `.env*`, reconciling *declared* configuration against *actual* wiring. It ships as a
standalone Solana power tool — `brainblast wallet-check [dir]` — matching the firewall / keys /
signguard pattern (standalone command + inline export), with an `allow / warn / block` verdict and an
exit code for CI.

## The four findings (one pass, env↔code reconciliation)

| id | sev | fires when |
|---|---|---|
| **`solana-wallet-network-mismatch`** (A) | 🔴 critical | `.env` declares one network (devnet/testnet/localnet) but the `ConnectionProvider` endpoint is hardcoded to a *different* concrete network (e.g. `clusterApiUrl(WalletAdapterNetwork.Mainnet)`, a literal `api.mainnet-beta.solana.com`) and isn't wired to the env var |
| **`solana-network-env-unwired`** (A′) | 🟠 high | a network env var is declared but **no source file references it** — the value is dead and the app ships whatever the source hardcodes (the exact bug that triggered this release) |
| **`solana-public-rpc-endpoint`** (B) | 🟠 high | the live endpoint is the **public mainnet RPC** (`clusterApiUrl('mainnet-beta')` / `api.mainnet-beta.solana.com`) — rate-limited, fails under real load. Only fires for mainnet-bound endpoints (devnet public RPC is fine for a demo) |
| **`solana-rpc-key-exposed`** (C) | 🟠 high | a client-exposed env var (`NEXT_PUBLIC_*` / `VITE_*` / `REACT_APP_*`) holds an RPC URL with an embedded API key (Helius/QuickNode/Alchemy/…) — it ships to every browser; anyone drains your paid quota |
| **`solana-wallet-ui-styles-missing`** (D) | 🟡 medium | uses `WalletModalProvider` / `WalletMultiButton` but never imports `@solana/wallet-adapter-react-ui/styles.css` — the connect modal renders **unstyled / looks broken** |

Verdict: any critical → `block` (exit 1); any high/medium → `warn`; none → `allow`. `--strict` makes
high fail too.

## Honesty guards (so we never overclaim)

- **B** only fires when the resolved network is mainnet — a devnet hello-world on the public RPC is fine.
- **D** is phrased "modal renders unstyled / may look broken," not "button does nothing."
- The mismatch finding is framed around the *verifiable* fact — *your endpoint/network doesn't match
  your configured env* — not a claim about Phantom's internal network selection (which depends on the
  wallet's own RPC and isn't statically visible).
- Detection is conservative: `solana-rpc-key-exposed` requires both a client-exposed prefix **and** a
  key-bearing URL; `network-env-unwired` only fires when wallet-adapter code is actually present.

## Surfaces

- **`brainblast wallet-check [dir]`** — the analysis pass; tiered report, `--json`, `--strict`, exit 1
  on block.
- **`inspectWalletConfig(dir)`** — inline export for an agent to call before shipping a frontend.
- Surfaces in the **research/predict** phase too: when a spec says "devnet", remind the agent to wire
  `NEXT_PUBLIC_SOLANA_NETWORK` into the adapter's `network`/endpoint.

## Build order

1. `src/wallet/envConfig.ts` — parse `.env*` for network + RPC vars, normalize the declared network,
   detect embedded RPC keys.
2. `src/wallet/codeConfig.ts` — scan wallet-adapter files for the endpoint network (clusterApiUrl /
   literal URLs / `WalletAdapterNetwork.X`), env-var wiring, modal UI usage, styles import.
3. `src/wallet/analyze.ts` — reconcile the two → findings + verdict + render.
4. CLI `wallet-check` + inline export + tests.

Standalone, content-based analysis (no ts-morph overhead, conservative regexes), fully offline and
unit-testable with temp dirs. Ships as a point release: `0.8.1 → 0.8.2`.
